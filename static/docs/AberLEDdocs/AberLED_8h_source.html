49 /// The class for the AberLED shield. One object of this class, called
50 /// **AberLED**, is automatically created (similar to how Serial works).
51 /// Yes, all this time you've been writing C++, not C. You should put
52 /// `AberLED.` before every function, so `AberLED.begin()` instead
53 /// of just `begin()`.
54 ///
55 /// \warning
56 /// If you are using a TFT display, you should call
57 /// `AberLED.begin(true)` instead!
58 ///
59 ///
60 /// Calling begin() will initialise the display, setting all the pins to
61 /// the appropriate mode and setting up the interrupt handler.
62 /// Thereafter, every nth of a second the Arduino will automatically
63 /// send the front buffer to the matrix.
64 /// Calling getBuffer() will give a pointer to the back buffer, which
65 /// can be be written to. Alternatively, pixels can be written by calling
66 /// set() with x,y coordinates and a colour (GREEN, RED, YELLOW or BLACK).
67 /// Once writing is complete, calling swap() will
68 /// exchange the front and back buffers, so that the Arduino will
69 /// now write the new front buffer to the matrix every interrupt tick.
70 ///
71 /// The format of the buffers is 8 16-bit integers: unsigned shorts.
72 /// Each integer represents a single row. Each pixel - individual LED -
73 /// is represented by 2 adjacent bits with the most significant bit
74 /// being red and the least significant bit being green.
75
